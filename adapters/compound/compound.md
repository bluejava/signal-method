# Compound Phase

Use this guidance when integrating the methodology into a compound `compound` phase.

## Objectives

- preserve reusable knowledge
- preserve decision history
- synchronize canonical docs with reality

## Suggested Sequence

1. Update canonical docs that changed.
2. Add or update ADRs for durable decisions.
3. Add compound-memory notes for reusable lessons.
4. Update `signal-docs/doc-index.md` or the equivalent doc index in the configured canonical docs root if the documentation graph changed.
5. Close or pause workflow state with resume instructions.
6. Confirm the next similar task would start from better context than this one did.

## Capture Rules

- Put current truth in canonical docs.
- Put reasoning in ADRs.
- Put reusable implementation knowledge in compound memory.
- Put workflow progress and next steps in workflow state.
- Do not dump everything into one “notes” file.
