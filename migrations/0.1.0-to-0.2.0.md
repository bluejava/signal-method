# Migration: 0.1.0 To 0.2.0

## Summary

Adds native Signal workflows, workflow state, methodology version metadata, and repositions compound engineering as an optional adapter.

## Detect

A project is probably pre-0.2.0 if it is missing either file:

- `signal-method.json` at the project root
- `workflow-state/current.md` under the configured docs root

Use the configured docs root. In most projects this is `signal-docs/`, but some projects may use `docs/`.

## Steps

1. Add `signal-method.json` at the project root.
   - Set `version` to `0.2.0` only after the rest of this migration is complete.
   - Set `docsRoot` to the actual docs root.
2. Add `workflow-state/current.md` if no workflow state exists.
3. Update `doc-index.md` to include `workflow-state/current.md`.
4. Update agent loading guidance so agents read active workflow state after the documentation index.
5. Update project agent guidance so agents update workflow state at the end of meaningful Signal workflow steps.
6. If the project has compound-specific guidance, reframe it as optional adapter guidance instead of the authoritative workflow.
7. Update workflow state with this migration's completed steps and next step.
8. Set `signal-method.json` to `0.2.0`.

## Do Not

- Overwrite canonical docs from the latest template.
- Rewrite feature specs unless the project is actively migrating a feature workflow.
- Delete compound files automatically.
- Rename the docs root during this migration.

## Completion Checks

- The project root has `signal-method.json`.
- The docs root has `workflow-state/current.md`.
- The documentation index points to workflow state.
- Agent guidance tells agents to maintain workflow state.
- Any compound guidance is compatibility-oriented.
- The recorded methodology version is `0.2.0`.
