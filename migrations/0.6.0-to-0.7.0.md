# Migration: 0.6.0 To 0.7.0

## Summary

Adds a generated project dashboard for human orientation.

Signal Method `0.7.0` introduces `project-dashboard.html` as a derived status overview generated from existing Signal artifacts. The markdown files remain canonical truth.

## When To Apply

Apply this migration when:

- `signal-method.json` records version `0.6.0`
- the project wants the standard Signal dashboard surface
- a human-facing overview would make project status easier to inspect

This migration is additive. It should not rewrite project-specific canonical docs unless the project chooses to add dashboard navigation links.

## Steps

1. Read `signal-method.json` and identify the configured docs root.
2. Ensure the project has `workflow-state/current.md`.
3. Run the dashboard generator from the installed Signal Method skill:

   ```bash
   node <skill-root>/scripts/generate-dashboard.js <project-root>
   ```

4. Confirm `<docsRoot>/project-dashboard.html` was created.
5. Optionally add a short link to `project-dashboard.html` in the project's `doc-index.md` or local onboarding docs.
6. If the project keeps agent loading guidance, optionally mention that humans can open the dashboard first for quick status orientation.
7. Set `signal-method.json` to `0.7.0` after the dashboard exists and local navigation guidance is reconciled.

## Avoid

- Treating `project-dashboard.html` as canonical truth.
- Hand-editing dashboard status instead of regenerating it.
- Overwriting project-specific docs with template copies.
- Requiring dashboard regeneration after every trivial agent interaction.

## Exit Criteria

- `<docsRoot>/project-dashboard.html` exists.
- The dashboard links back to source markdown artifacts.
- Any local docs that should mention the dashboard have been updated.
- The recorded methodology version is `0.7.0`.
