# Migration: 0.3.0 To 0.4.0

## Summary

Adds roadmap planning as a native Signal workflow and introduces two optional canonical planning artifacts:

- `product-goals.md`
- `roadmap.md`

Use this migration when a project wants Signal to support big-picture planning before feature work.

## Detect

A project is probably on 0.3.0 if:

- `signal-method.json` records version `0.3.0`
- the docs root has no `product-goals.md`
- the docs root has no `roadmap.md`
- agent guidance or workflow references only mention initialization, migration, feature work, and methodology upgrade

## Steps

1. Read the project's `signal-method.json` and confirm the configured docs root.
2. Update workflow state to record the methodology upgrade.
3. Add `product-goals.md` to the docs root if big-picture product planning should be durable in this project.
4. Add `roadmap.md` to the docs root if goals need to be broken into phases, iterations, releases, or milestones.
5. Update `doc-index.md` to list the new planning artifacts under the canonical system model.
6. Update project agent guidance if it names the available Signal workflows.
7. Use `workflows/plan-roadmap.md` in the methodology source repo, or the Roadmap Planning section of `references/workflow-map.md` in the installed skill, for the first planning pass before creating or updating feature specs.
8. Update workflow state with completed steps and the next planning or build-feature step.
9. Set `signal-method.json` to `0.4.0` after the migration is complete.

## Do Not

- Invent product goals while migrating if the project owner has not provided enough direction.
- Treat the roadmap as a commitment schedule unless the project explicitly uses it that way.
- Put detailed feature behavior in `roadmap.md`; use `feature-specs/` for buildable behavior.
- Replace existing project planning docs without reconciling their current truth into the new artifacts.

## Completion Checks

- `doc-index.md` points to any planning artifacts that were added.
- `workflow-state/current.md` records the migration result and next step.
- Selected roadmap items point toward feature specs before entering `build-feature`.
- The recorded methodology version is `0.4.0`.
