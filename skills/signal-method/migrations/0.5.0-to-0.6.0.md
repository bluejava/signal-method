# Migration: 0.5.0 To 0.6.0

## Summary

Adds explicit roadmap alignment to meaningful sibling workflow state files.

Signal Method `0.6.0` requires workflow state to explain how active work relates to `roadmap.md`, while preserving flexibility for enabling work, maintenance, and roadmap gaps.

## When To Apply

Apply this migration when:

- `signal-method.json` records version `0.5.0`
- the project has active or paused sibling workflow state files under `workflow-state/`
- the project uses `roadmap.md`, plans to use roadmap planning, or wants workflow-to-roadmap traceability

Closed historical workflow state files may be left unchanged unless they are still used for handoff.

## Steps

1. Read `workflow-state/current.md` and identify active or paused sibling workflow state files.
2. For each meaningful active or paused sibling workflow state file, add a `Roadmap Alignment` section.
3. Use one of these alignment types:
   - roadmap phase
   - feature candidate
   - enabling work
   - roadmap gap
   - maintenance
   - not applicable
4. Add links to `roadmap.md`, `product-goals.md`, and any relevant feature spec when they exist.
5. Record how the workflow advances or affects the roadmap.
6. Record whether `roadmap.md` needs an update before the workflow closes.
7. If the workflow exposes missing roadmap structure, update `roadmap.md` or explicitly record why the roadmap should remain unchanged.
8. Update `workflow-state/current.md` index entries with a short roadmap alignment note when useful for routing.
9. Update `doc-index.md` if it lists methodology version-specific artifacts or migration guides.
10. Set `signal-method.json` to `0.6.0` after the migration is complete.

## Suggested Section

```md
## Roadmap Alignment

- Alignment type:
- Roadmap link:
- Product goal link:
- Feature spec link:
- How this workflow advances or affects the roadmap:
- Roadmap update needed before close:
```

## Avoid

- Forcing every workflow to equal exactly one roadmap phase.
- Expanding `roadmap.md` into a task backlog for every implementation detail.
- Leaving roadmap-adjacent work unclassified.
- Updating version metadata before the workflow state files are reconciled.

## Exit Criteria

- Active and paused sibling workflow state files include `Roadmap Alignment`.
- Roadmap gaps are either reflected in `roadmap.md` or explicitly deferred.
- `workflow-state/current.md` still acts as an index.
- The recorded methodology version is `0.6.0`.
