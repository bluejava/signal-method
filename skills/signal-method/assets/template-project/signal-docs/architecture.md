# Architecture

Purpose
Describe the technical structure of the system.

Include

- major components
- responsibilities
- data flow
- runtime boundaries
- architectural constraints

Exclude

- feature-specific behavior
- historical decision rationale
- visual design rules

Split when
Subsystems become large enough that combining them hurts scanning or retrieval.

## Components

Describe each major component and its responsibility.

## Data Flow

## Runtime Environments

## Constraints
