# System Overview

Purpose
Describe the conceptual model of the system so a human or AI can understand what the product is and where its boundaries are.

Include

- product purpose
- core concepts
- major subsystems
- system boundaries
- a compact mental model of how the pieces fit together

Exclude

- detailed implementation
- API contracts
- schema details
- visual design rules

Split when
The product has multiple large conceptual domains that are often understood separately.

## Product Purpose

Describe what the product exists to do.

## Core Concepts

List the key domain concepts and terms.

## Major Subsystems

Describe the major pieces at a high level.

## System Boundaries

Explain what is inside the system and what is external.
