# ADR-001 Example Decision

Purpose
Capture why an important decision was made.

Include

- context
- decision
- alternatives considered
- consequences

Exclude

- long implementation tutorials
- general feature descriptions
- current truth that belongs in canonical docs

Split when
Do not split one decision across multiple ADRs unless they are truly independent.

## Status

Accepted

## Context

Describe the situation that required a decision.

## Decision

State the decision clearly in one or two sentences.

## Alternatives Considered

1. Alternative A
2. Alternative B

## Consequences

- positive consequence
- tradeoff
- follow-up implication
