# Template Project

Copy the `signal-docs/` folder into a real project and replace the example content with project-specific material.

Open `signal-docs/project-dashboard.html` when you want a quick generated orientation snapshot. Regenerate it from the installed Signal Method skill after meaningful workflow updates when a fresh human-facing status view is useful.

Use the templates as constraints, not ceremony:

- fill the smallest useful set of files first
- keep broad docs stable
- move volatile detail into local artifacts
- split only when retrieval or maintenance clearly improves
