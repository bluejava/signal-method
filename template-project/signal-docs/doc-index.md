# Documentation Index

Purpose
Provide the entry point to the project documentation graph.

Include

- the main docs grouped by retrieval purpose
- short descriptions of each file or folder
- navigation hints for new contributors and AI tools

Exclude

- detailed content duplicated from the target docs
- long historical narrative

Split when
Usually do not split this file. Keep it concise and central.

## Canonical System Model

- `system-overview.md`: conceptual model and boundaries.
- `product-goals.md`: big-picture direction, priorities, non-goals, and success signals.
- `roadmap.md`: phases, iterations, releases, or milestones and feature candidates.
- `system-invariants.md`: rules that must always remain true.
- `architecture.md`: technical structure and component responsibilities.
- `data-model.md`: shared entities and relationships.
- `api.md`: interfaces and contracts.
- `design/design-guidelines.md`: UX, interaction, and presentation consistency.
- `feature-specs/`: feature-level behavior.

## Decision History

- `adr/`: important design choices and their rationale.

## Compound Knowledge

- `compound-memory/`: reusable lessons, pitfalls, and solution notes.

## Process And Agent Behavior

- `agent-guidance/`: project-specific operational guidance for AI and human contributors.
- `workflow-state/current.md`: workflow-state index and resume routing.
