# Data Model

Purpose
Define the shared entities and relationships used across the system.

Include

- core entities
- relationships
- lifecycle notes
- data invariants

Exclude

- endpoint documentation
- architecture narrative
- feature walkthroughs

Split when
The model spans multiple coherent domains that are usually retrieved separately.

## Shared Vocabulary

List the canonical names of the main entities.

## Entities

Document fields, responsibilities, and identity rules for each entity.

## Relationships

Describe how the entities connect.

## Invariants

Record the rules that must remain true for the data model.
