# Example Feature

Purpose
Describe the behavior of a feature or feature area.

Include

- feature behavior
- inputs and outputs
- user or system flows
- constraints
- interactions with system components

Exclude

- global architecture rules
- ADR rationale
- reusable implementation lessons

Split when
A feature becomes large enough to need supporting files or a subfolder.

## Summary

Describe what the feature does and why it exists.

## Actors

List the humans or systems involved.

## Inputs

Describe what enters the feature.

## Outputs

Describe what the feature produces.

## Main Flow

1. User or system initiates the feature.
2. The system validates inputs.
3. The feature performs its core behavior.
4. The system returns or emits the result.

## Constraints

List the conditions the implementation must respect.

## System Interactions

Explain which parts of the broader system this feature touches.

## Open Questions

List unresolved issues separately from current truth.
