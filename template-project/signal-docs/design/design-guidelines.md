# Design Guidelines

Purpose
Define product consistency rules for interaction, presentation, and content.

Include

- interaction conventions
- layout patterns
- wording guidance
- visual rules
- established product precedents worth preserving

Exclude

- system architecture
- feature-level behavior specs
- implementation code as the main content

Split when
Visual, interaction, and content guidance become large enough to retrieve separately.

## Interaction Patterns

Document the important UX precedents the product should preserve.

## Layout And Components

Describe recurring layout and component usage rules.

## Content And Tone

Describe wording, labels, and tone guidance.

## Visual Rules

Describe spacing, hierarchy, iconography, motion, or other visual consistency rules.
