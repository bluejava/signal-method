# Example Pattern

Purpose
Capture a reusable lesson from implementation work.

Include

- the problem
- the solution pattern
- the conditions where it applies
- pitfalls or edge cases

Exclude

- canonical system rules
- architecture definitions
- one-off chatter with no reuse value

Split when
Patterns are consulted separately often enough to justify domain-specific grouping.

## Problem

Describe the recurring problem or scenario.

## Pattern

Describe the reusable approach.

## Why It Works

Explain the underlying reason the pattern is effective.

## Pitfalls

List the cases where the pattern fails or needs adjustment.

## Reuse Signals

Explain how to recognize similar future situations.
